## High‑Level Architecture

- The application is a **Vue 3** frontend using **TypeScript** and **Vite** as the bundler.
- **Routing** lives in `src/router/index.ts` and defines the main navigation structure.
- **State & Types**: Shared TypeScript types are defined under `src/types/*.ts`. The `settings` type (`src/types/settings.ts`) describes user‑configurable options.
- **Configuration**: Runtime configuration is loaded from JSON files in `public/` (`settings.json`, `settings.json.tpl`).
- **Docker**: The repository includes a `Dockerfile` and a `docker/entrypoint.sh` script for containerised builds.
- **Testing**: Vitest is configured via `vitest.config.ts`. Tests follow the `*.spec.ts` naming convention under `src/`.
- **Linting/Formatting**: ESLint and Prettier configurations are expected to be present (standard for Vue 3 projects).

## Development Tips

- Run `pnpm run lint` before committing to catch style issues early.
- Use the `Makefile` targets for common tasks (`make test`, `make lint`, `make build`).
- When adding new components, place them under `src/components/` and export them via an index file for easy imports.
- For any new TypeScript types, add them to the appropriate file in `src/types/` and update imports accordingly.
